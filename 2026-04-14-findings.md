## [2026-04-14] Microsoft Patch Tuesday April 14, 2026 - 80-100+ vulnerabilities + Secure Boot certificate expiration deadline

- **Source**: [Zecurit Patch Tuesday Analysis](https://zecurit.com/endpoint-management/patch-tuesday/) / [Help Net Security April Forecast](https://www.helpnetsecurity.com/2026/04/10/april-2026-patch-tuesday-forecast/) / [LayerLogix Critical Patches Guide](https://layerlogix.com/blog/2026/04/april-2026-patch-tuesday-critical-vulnerabilities-houston-it-deployment-guide) / [Microsoft Security Response Center](https://msrc.microsoft.com/update-guide/)
- **Confidence**: Alta
- **What changes**: Microsoft released the April 2026 Security Update today (April 14, 2026) with **80-100+ security vulnerabilities** patched across Windows, Office, and server components. Highlights: Focus on Windows kernel security, Office applications (Excel, Word, Outlook, Teams integration), and cloud services. Key new features mixed with security hardening for Windows 11 (v24H2 and v25H2). Most critical non-CVE item: **Secure Boot certificates issued in 2011 expire June 26, 2026** (72 days from today) — Windows Server instances require **manual action before June 26** to avoid boot failures, as the legacy Secure Boot certificates will no longer be trusted during the boot process. **No automatic remediation available** — requires manual cert updates or BIOS/UEFI firmware updates.
- **Action required**: Urgente
- **Details**: Deploy April 14 patches via Windows Update or WSUS. The Secure Boot deadline (June 26) is the more critical operational issue: organizations must audit systems running Windows Server with Secure Boot enabled and verify firmware supports new certificates or implement manual boot key updates. Affected systems: Windows Server 2016+, Windows 10/11 with Secure Boot enabled. Failure to patch before June 26 will brick servers on next restart if they attempt Secure Boot validation. Check firmware vendor (Dell, HP, Lenovo, etc.) for BIOS updates that include the new Secure Boot certificates, or disable Secure Boot if not required in the environment. Organizations with automated boot procedures should test patches in staging before rolling out to production — the March/April 2026 Patch Tuesday cycle has seen multiple installation failures reported.

# Hallazgos de Investigacion - 3 Mayo 2026

## [2026-05-03] Node.js 20 EOL deadline PASÓ - 3 dias en atraso

- **Fuente**: [Node.js End-of-Life oficial](https://nodejs.org/en/about/eol) / [Dev Community](https://dev.to/matheus_releaserun/nodejs-20-end-of-life-migration-playbook-for-april-30-2026-2onh)
- **Confianza**: Alta
- **Que cambia**: Node.js 20 alcanzó EOL el 30 de abril 2026. Hoy es 3 de mayo, pasó el deadline. Node.js ya NO proporciona security patches para la rama 20.x. Si se descubre un CVE despues de hoy, solo se parchará Node 22+.
- **Accion requerida**: URGENTE si aun se usan servidores en Node 20
- **Detalles**: En AWS Lambda, ya no se pueden crear nuevas funciones en nodejs20.x desde 1 de junio 2026. Las funciones existentes perderán soporte tecnico. Migrar inmediatamente a Node 22 LTS o Node 24. Opciones de extended support disponibles via HeroDevs, TuxCare.

## [2026-04-08] CVE-2026-23869 - React Server Components DoS vulnerability CRITICA

- **Fuente**: [React Blog oficial](https://react.dev/blog/2025/12/11/denial-of-service-and-source-code-exposure-in-react-server-components) / [Vercel Changelog](https://vercel.com/changelog/summary-of-cve-2026-23869) / [NVD](https://nvd.nist.gov/vuln/detail/CVE-2026-23869)
- **Confianza**: Alta
- **Que cambia**: Vulnerabilidad DoS de alta severidad (CVSS 7.5) en React Server Components. Requests HTTP especialmente crafted pueden trigger excessive CPU usage en endpoints App Router Server Function. Afecta react-server-dom-parcel, react-server-dom-turbopack, react-server-dom-webpack versiones 19.0.0, 19.1.0, 19.2.0.
- **Accion requerida**: URGENTE - parchear
- **Detalles**: Versiones parcheadas: 19.0.5, 19.1.6, 19.2.5. Este CVE es especialmente peligroso para servidores que usen React Server Components en produccion. Implementar rate limiting en Server Function endpoints como mitigation temporal. Actualizar a versiones parcheadas inmediatamente.

## [2026-04-13] React Router v7.14.2 - breaking changes en RSC Framework Mode

- **Fuente**: [React Router Changelog oficial](https://reactrouter.com/changelog) / [Releasebot](https://releasebot.io/updates/remix/react-router)
- **Confianza**: Alta
- **Que cambia**: React Router v7.14.0+ require cambios breaking para proyectos que adoptaron RSC Framework Mode (unstable). Las route exports deben cambiar: `default` → `ServerComponent`, `ErrorBoundary` → `ServerErrorBoundary`, `Layout` → `ServerLayout`, `HydrateFallback` → `ServerHydrateFallback`. Las route modules que NO fueron prefijadas con "Server" ya no son implicitamente server components.
- **Accion requerida**: Actualizar dependencia si se usa RSC Framework Mode
- **Detalles**: Este es un breaking change SOLO para proyectos que ya usan RSC Framework Mode en estado unstable. La mayoria de aplicaciones React Router standar no son afectadas. Cambio fue necesario para soportar SSR y pre-rendering de forma consistente.

## [2026-04-27] Node.js 24 entra en LTS - soporte extendido

- **Fuente**: [Node.js Release Info](https://nodejs.org/en/about/eol)
- **Confianza**: Alta
- **Que cambia**: Node.js 24 entró oficialmente en Long-Term Support (LTS) con version 24.11.0 (27 abril 2026). Soporte durará hasta 30 abril 2028. Node 24 ahora es opcion viable para produccion a largo plazo.
- **Accion requerida**: Monitorear
- **Detalles**: La entrada de Node 24 a LTS proporciona alternativa de upgrade path para quienes salen de Node 20. Ventajas: mas reciente que Node 22, mejor performance, features mas nuevas. Desventajas: menos testing en wild que Node 22. Recomendacion: migrar a Node 22 si necesitas estabilidad conocida, o Node 24 si aceptas ser early en LTS cycle.

## [2026-05-03] CVE-2025-55182 - React Server Components RCE crítico (React2Shell)

- **Fuente**: [React Blog oficial](https://react.dev/blog/2025/12/03/critical-security-vulnerability-in-react-server-components) / [Microsoft Security Blog](https://www.microsoft.com/en-us/security/blog/2025/12/15/defending-against-the-cve-2025-55182-react2shell-vulnerability-in-react-server-components) / [Palo Alto Unit 42](https://unit42.paloaltonetworks.com/cve-2025-55182-react-and-cve-2025-66478-next/) / [Google Cloud Blog](https://cloud.google.com/blog/topics/threat-intelligence/threat-actors-exploit-react2shell-cve-2025-55182)
- **Confianza**: Alta
- **Que cambia**: CVE-2025-55182 es una RCE pre-autenticada de CVSS 10.0 en React Server Components. El protocolo Flight deserializa insecuramente payloads atacante, permitiendo ejecucion de codigo arbitrario en el servidor. Afecta React 19.0-19.2 (arreglado en 19.0.1, 19.1.2, 19.2.1) y Next.js 14.3.0-canary.77+, 15.x y 16.x. Explotacion activa confirmada desde 5 de diciembre 2025, incluyendo threat actors entregando coin miners.
- **Accion requerida**: Urgente
- **Detalles**: CRITICAL - actualizar React a 19.0.1+, 19.1.2+, o 19.2.1+. Actualizar Next.js a versiones parcheadas. El ataque es trivial: un POST request malformado a cualquier Server Functions endpoint causa RCE. Google GTIG observo explotacion generalizada por multiplos threat clusters. No hay workaround sin parchear - requiere actualizacion de dependencias.

## [2026-05-03] Next.js 15 - Breaking changes en caching y async request APIs

- **Fuente**: [Next.js Blog oficial](https://nextjs.org/blog/next-15) / [Next.js Upgrade Guide](https://nextjs.org/docs/app/guides/upgrading/version-15)
- **Confianza**: Alta
- **Que cambia**: Next.js 15 introduce dos breaking changes principales: (1) Async Request APIs ahora requeridas para headers(), cookies(), searchParams; (2) fetch requests, GET Route Handlers y client navigations no se cachean por defecto (antes si). El minimo Node.js requerido es 18.18.0.
- **Accion requerida**: Monitorear
- **Detalles**: La semantica de caching cambia significativamente. Codigo que dependia de caching implicito en handlers GET fallara silenciosamente. Async request APIs requieren cambios en componentes que usan headers/cookies. Next.js proporciona codemods para automatizar parte de la migracion. Las nuevas features: React 19 support, Turbopack stable, `next/form`, Server Actions security improvements.

## [2026-05-03] CVE-2026-23864 - DoS en React/Next.js Server Functions

- **Fuente**: [Akamai Security](https://www.akamai.com/blog/security-research/cve-2026-23864-react-nextjs-denial-of-service) / [Vercel Changelog](https://vercel.com/changelog/summary-of-cve-2026-23864)
- **Confianza**: Alta
- **Que cambia**: Vulnerabilidad DoS complementaria a CVE-2025-55182. POST requests malformados a Server Functions endpoints pueden causar infinite loops, memory exhaustion o excessive CPU. No causa RCE como CVE-2025-55182 pero deshabilita el servidor.
- **Accion requerida**: Urgente
- **Detalles**: Mismas versiones afectadas: React 19.x, Next.js 14.3+ canary, 15.x, 16.x. Mismo fix: actualizar a versiones parcheadas. Ambos CVEs atacan el protocolo Flight con inputs crafted. Importante parchear ambas vulnerabilidades.
